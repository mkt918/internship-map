# インターンシップ管理

## 作業概要（v1）
インターンシップ事業所・生徒データをもとに、地図表示とデータベース検索ができるWebページを作成。

## 作業概要（v2）
配置最適化スクリプトで全生徒の行き先を新規割り当て。地図用データベースCSVを出力。

## 使ったインプット
- `input/260617_インターンシップ事業所.csv` — 事業所49件（No・企業名・業種・住所・連絡先など）
- `input/260617_クラス名簿表フル.csv` — 生徒144件（学年・組・番号・I/L業種・P/R/T希望・最寄り駅・大会重複など）

## アウトプット
- `index.html` — インターンシップ管理Webアプリ（GitHub Pages公開済み）
- `assign.py` — 配置最適化Pythonスクリプト
- `新配置案.csv` — 生徒144件の新配置データベース（地図用）

## 新配置案.csv の仕様
列: 学年, 組, 番号, 性別, 配属企業No, 企業名, 業種, 受け入れ先住所, 緯度, 経度, 日付, 最寄り駅, 市町, 距離km, 業種マッチ, ①業種(I), ②業種(L), 第1希望(P), 第2希望(R), 第3希望(T), 大会重複, 備考

業種マッチ判定:
- ✅IL一致: I列またはL列の業種が企業業種と一致（最優先）
- ⭕第1～3希望: P/R/T希望業種で一致
- 🔴不一致: 業種マッチなし（今回は発生せず）

## 配置結果サマリ

- ✅IL一致: 129件
- ⭕第2希望: 13件
- ⭕第3希望: 2件
- 1年生(40名)全員7月配置 ✓
- 2年生: 7月69名 / 8月35名
- 定員超過: 0件

## 配置アルゴリズム概要
1. フェーズ1: 1年生 → 4優先企業(42B/43B/2C/2D)へ業種I/L一致生徒を優先配置
2. フェーズ2: 残り全員を「利用可能な選択肢が少ない生徒から順」に配置
   - 優先順: I/L一致 > 第1希望 > 第2希望 > 第3希望 > 業種不一致
   - 同レベルは自宅最寄り駅からのHaversine距離で近い方優先
   - 1年生=7月のみ、2年生=7月優先→枠なし時は8月
   - 大会重複日を自動除外
3. 定員管理: 日別定員 + 特殊制約（「1日のみ」「合計N名」「前半/後半別」）

## 特記事項（v1）
- 座標は知識ベースで直接埋め込み（GSI APIは名称曖昧で不正確）
- 企業クリックで生徒の自宅最寄り駅から直線を地図上に描画

## 作業概要（v4）
生徒名簿DBに「事前説明1・2」列を追加し、これを最優先にした新マッチ階層で再配置。

## 使ったインプット（v4）
- `企業マスターDB.csv` — 49社（業種カテゴリ: 保育/製造工業/製造食品/物流/美容/販売）
- `生徒名簿DB.csv` — 144名。E/F列「事前説明1・2」を追加、正規化カテゴリ列「事前説明1業種・2業種」も付与

## アウトプット（v4）
- `assign2.py` — 正規化DBを直接読む新配置スクリプト
- `新配置案.csv` — 144名再配置（列: …業種マッチ, 事前説明1, 事前説明2, 第1〜3希望）

## マッチ階層（v4・確定）
事前説明1 → 事前説明2 → 第1希望 → 第2希望 → 第3希望 → 不一致
（事前説明の正規化: 製造（食品）=製造食品 / 製造（工業）=製造工業 / 販売・サービス・アパレル=販売 / 保育・幼児教育=保育 / 運輸・物流=物流 / 美容=美容）

## 配置方針（v4・ユーザー確認済み）
- バランス型（業種マッチ優先＋同マッチ度なら近い企業）
- 原則15km・例外可
- 特殊ルール全適用（1年生→4優先企業 / 3D→北名古屋物流希望 / 保育希望なし→製造系先行）
- 競合時は選択肢が少ない生徒を優先

- 事前説明で73%（105名）が充足
- 15km超過: 1件（3D物流優先の例外）/ 定員超過: 0

## 作業概要（v3）
GitHub Pages で動く手動配置編集Webサイトを作成。

## アウトプット（v3）
- `assign_manual.html` — 手動配置ツール（GitHub Pages 公開用）
  - 企業マスターDB.csv / 生徒名簿DB.csv を自動フェッチまたはファイルアップロードで読み込み
  - 左パネル: 事業所一覧（業種・学年フィルタ、配置数/定員表示）
  - 中央パネル: 選択事業所の日程タブ＋配置スロット
  - 右パネル: 生徒一覧（✅⭕🔴マッチ表示、学年・組・ステータスフィルタ）
  - 「事業所選択 → 日付タブ選択 → 生徒クリック」で配置
  - 既存配置（新配置案.csv）の読み込み機能
  - LocalStorage に自動保存
  - 配置案.csv エクスポート

## 特記事項（v3）
- CSVパーサーをcharacter-by-charに書き直し（複数行クォートフィールド対策）
- 業種マッチは第1〜3希望業種とcompany業種カテゴリの比較（製造工業・製造食品は相互マッチ）
- v3マッチ判定はkjファイルのP/R/T希望のみ（assign.pyのI/L一致とは別ロジック）

## 特記事項（v2）
- 41C(郵便局)など「いずれかの1日のみ」の会社は total_cap を3に設定
- 43B(きもの館)は「前半6名・後半6名」の月別上限を設定
- 2年4組の一部生徒は製造食品系を希望するも枠埋まりのため第2希望(販売/保育系)に配置
- 今後HTMLを更新して旧配置と新配置を並べて確認できるようにすることも可能
